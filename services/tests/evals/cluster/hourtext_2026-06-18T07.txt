  ⏺ Here are all the non-summary findings from the research, grouped by whether they help.
  Things that could help (but secondary)
  1. Model quantization — 3-bit DWQ
  - Decode is memory-bandwidth-bound, so fewer bits = faster (4-bit ≈ 4× fp16). Going to 3-bit is ~⅓ faster decode.
  - Plain 3-bit has a real quality cliff (logit similarity drops 0.995
  3-bit speed with ~4-bit quality. DWQ does not change runtime speed by itself; it only buys quality headroom to drop a bit-level.
  - Verdict: the best per-token speedup lever, but needs an A/B through the 103 goldens before swapping the production model.
  2. Prefix / prompt caching
  - Big correction: the cache being "non-trimmable" is not because it's 4-bit. QuantizedKVCache is trimmable; only sliding-window/hybrid caches (RotatingKVCache/SSM) aren't.
  - And trimmability is a red herring for reuse anyway — you can prefill the static system+skill prefix once and deepcopy it per call (append-only, no trimming needed).
  Officially supported pattern.
